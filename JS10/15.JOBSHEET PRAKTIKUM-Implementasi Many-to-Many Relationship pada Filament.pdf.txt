     B. Permasalahan Penyimpanan Tag 
Sebelumnya tag disimpan dalam format JSON di tabel posts. 
Contoh: 
tags 
--------- 
["Laravel 12","PHP"] 
Kelemahan metode ini: 
• Data sulit dimodifikasi 
• Duplikasi data 
• Tidak efisien untuk query database 
• Tidak terstruktur 
Solusi: 
Menggunakan Many-to-Many Relationship dengan tabel pivot.

     D. Rollback Migration 
Jika sebelumnya menggunakan JSON column pada tags, rollback migration terlebih dahulu. 
• php artisan migrate:rollback 
 
note : kita akan melakukan rollback pada tabel prodect dan post, Untuk menentukan berapa kali 
rollback, kita lihat urutan timestamp migration karena Laravel rollback berdasarkan batch terakhir 
yang dijalankan. Sebagai contoh dari folder migrations 

 
Jika ingin rollback posts dan products 
Yang terakhir dibuat: 
• products 
posts 
Maka harus rollback 2 langkah. 
Kemudian hapus kolom: 
• $table->json('tags'); pada create_posts_table.php 
 
 
     E. Membuat Tabel Tags 
Buka file create_posts_table.php dan tambahkan kode pada migration: 
• Schema::create('tags', function (Blueprint $table) { 
    $table->id(); 
    $table->string('name'); 
    $table->timestamps(); 
}); 

 
 
     F. Membuat Pivot Table 
Buka file create_posts_table.php dan tambahkan kode pada migration: 
• Schema::create('post_tag', function (Blueprint $table) { 
    $table->foreignId('post_id') 
        ->constrained() 
        ->cascadeOnDelete(); 
    $table->foreignId('tag_id') 
        ->constrained() 
        ->cascadeOnDelete(); 
    $table->primary(['post_id','tag_id']); 
}); 

 
 
Jalankan Migration 
• php artisan migrate

     I. Menambahkan Relationship pada Model Post 
Buka file post.php pada model dan hapus tags pada fillable 
 

Tambahkan function pada model post 
• public function tags() 
{ 
    return $this->belongsToMany(Tag::class,'post_tag'); 
} 
 
 
     J. Menambahkan Relationship pada Model Tags 
Tambahkan fungsi pada Model Tag 
• public function posts() 
{ 

    return $this->belongsToMany(Post::class,'post_tag'); 
} 
 
 
     K. Menggunakan Relationship pada Form Post 
Buka PostResource → Form Schema 
Ganti field tags dengan: 
• Select::make('tags') 
    ->relationship('tags','name') 
    ->multiple() 
 
Penjelasan: 
Method Fungsi 
relationship mengambil data relasi 
multiple memilih banyak data

     M. Membuat Relationship Manager
